Writing-style policy on Claude for Mac

Everything from the Cowork tab, no terminal required.

This manual is for someone who uses Claude for Mac and works in the Cowork tab. It covers installing the style toolkit, creating and deploying an edgar writing style, revising it, keeping several and switching between them, updating the plugin, and removing one. It assumes no terminal and no command-line knowledge; you do everything by talking to Claude.

Two ground rules to read first

Two kinds of slash command

A skill command begins with /edgar-style-policy: and works when you type it into the chat.

A plugin command, /plugin or /reload-plugins, does not work in Cowork; it belongs to the terminal. Wherever this manual needs one, you ask Claude in plain words, and Claude runs it for you.

Changes load when a session starts

After anything that installs, updates, removes, deploys, or switches, fully quit Claude for Mac (⌘Q) and open it again.

There is no in-app reload in Cowork. Skip this and nothing you just did will take effect; it is the most common reason a step seems not to work.

When Claude offers a choice of tier, choose the user tier. It needs no password and Claude does all of it from Cowork. The managed tier is sturdier against tampering, but installing or removing it requires a password command in the macOS Terminal, which a Cowork workflow avoids. This manual uses the user tier throughout.

1

Install the toolkit

The toolkit comes in two parts: a marketplace, the catalogue Claude reads from, and the plugin itself, named edgar-style-policy. You add both by asking Claude, once.

  1. Open a Cowork chat and ask Claude, plainly:
    "Please set up a Claude Code plugin for me. Add the plugin marketplace from the GitHub repository garycoding/claude-style-toolkit, then install the edgar-style-policy plugin from it."
    Naming it as a Claude Code plugin and a GitHub repository is what lets Claude recognize garycoding/claude-style-toolkit as an address it can add; Claude then runs the plugin commands for you and confirms them. If Claude seems unsure what to do, give it these two lines to run: 
    claude plugin marketplace add garycoding/claude-style-toolkit
claude plugin install edgar-style-policy@claude-style-toolkit
  2. Confirm it: in a new chat, type /edgar-style-policy: and you should see the toolkit's commands offered. Or ask Claude whether the plugin is installed, and which version.
Then: quit (⌘Q) and reopen.

Do not type /plugin into the app to install; it will not work there. The plain-language request is normally all it takes; the two fallback lines are only for the case where Claude does not recognize it.

2

Create and deploy your first style

An edgar writing style is a set of rules for how Claude writes: its voice, the words it must avoid, how it formats, how it handles claims. The style-author skill walks you through building one and then deploys it.

  1. In a Cowork chat, type /edgar-style-policy:style-author (or ask Claude to help you create an edgar writing style).
  2. Answer Claude's questions: the voice you want in a few adjectives, what past writing has done wrong, words to ban, formatting preferences, and a name. If you have documents whose voice you like, offer them; they are the strongest material Claude can work from.
  3. Claude drafts the style, reviews it with you, and refines it until you are satisfied. Take your time here; this is the part that matters.
  4. When Claude asks which tier, choose the user tier. Claude installs it with no password and saves a copy in your local library for reuse.
  5. Confirm it is active: ask Claude which writing style is active. From now on Claude follows your edgar writing style when it writes in Cowork.
Then: quit (⌘Q) and reopen before the style takes effect.
3

Revise an edgar writing style

To change a style you have already made, its voice, a word to ban, a formatting habit, use the style-maintain skill. It edits the style in place and redeploys it; you do not start over.

  1. In a Cowork chat, type /edgar-style-policy:style-maintain (or ask Claude to update your edgar writing style).
  2. Tell Claude what to change. The most useful inputs are concrete: a phrase it should stop using, an example document whose voice is right or wrong, or a specific thing it wrote that you did not want.
  3. Claude revises the style with you, one change at a time, until you are satisfied, then redeploys it with no password.
Then: quit (⌘Q) and reopen; the revision takes effect next session.

This changes an existing style. To move between styles you have stored, see the next section; to update the plugin software itself, see "Update the plugin."

4

Keep several styles and switch between them

Every style you author is kept in a local library, so you can build more than one and move between them by name. Nothing you switch away from is lost. To create a second style, run /edgar-style-policy:style-author again with a different name; it joins the first in your library.

  1. In a Cowork chat, type /edgar-style-policy:style-switch. Claude lists the styles in your library and marks which one is active.
  2. Tell Claude which style to switch to, by name.
  3. Claude re-deploys the chosen style for you, again with no password.
Then: quit (⌘Q) and reopen; the new style takes effect on the next session.

Switching back is the same procedure. Your styles stay in the library until you deliberately remove them, so you can move between them as often as you like.

5

Update the plugin

When a new version of the toolkit is published, the self-update skill brings you to it. This exists because the ordinary install command does not upgrade a plugin that is already installed; self-update does the whole thing for you.

  1. In a Cowork chat, type /edgar-style-policy:self-update.
  2. Claude refreshes the catalogue, checks your installed version against the latest, and, if a newer one exists, updates it for you and names the new version and what it adds. No password is needed.
Then: quit (⌘Q) and reopen to load the new version.

If Claude reports you are already on the latest version, there is nothing to do.

6

Remove a style, or remove the plugin

"Uninstall" can mean two different things; here is each.

To remove a writing style but keep the toolkit, use the style-uninstall skill:

  1. In a Cowork chat, type /edgar-style-policy:style-uninstall and confirm when Claude asks. Claude removes the active style's deployed files, with no password.
Then: quit (⌘Q) and reopen; Claude no longer applies the style.

Your library copy of that style is left in place, so you can switch back to it or redeploy it later; removing a style does not erase it from your collection.

To remove the toolkit itself, ask Claude in plain words: "Please uninstall the edgar-style-policy plugin." Claude runs the removal; then quit and reopen. This takes the plugin off your machine but still leaves your saved styles in the library untouched. If you want those gone as well, ask Claude to delete the folder ~/.claude/edgar-style-policies.

The one habit that makes it all work

After you install, deploy, switch, update, or remove anything, quit Claude for Mac with ⌘Q and open it again. Everything else is a matter of typing an /edgar-style-policy: command or asking Claude in plain words, and letting Claude do the work. You never need a terminal, and you should never be asked for a password; if you are, that is the managed tier, which you can decline in favour of the user tier this manual uses.
